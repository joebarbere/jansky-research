# Summary

`jansky-research` is a tested, CPU-first Python toolkit for reproducible
radio-astronomy analyses of public survey data, with optional, opt-in GPU
acceleration for its signal-processing and machine-learning components. It is the
research sibling of the `jansky` teaching course [@jansky], composing that course's
tested helpers rather than reimplementing them. It ships more than forty
self-contained analysis "slices", each built to the same discipline — a tested tool
(pure NumPy/SciPy/Astropy [@astropy2022], with a mandatory synthetic offline
fixture) run on a real public dataset, put through an adversarial science-review
gate, and written up as a short AASTeX paper whose every quantitative result is
generated by the pipeline rather than typed by hand. The `README` carries the
authoritative slice-by-slice inventory; the toolkit's reach, grouped by domain with
a few exemplars each, is:

- **Fast radio bursts & the time domain** — burst-population statistics on the First
  CHIME/FRB Catalog [@chimecat1] (`frbstats`), a phase-folding $Z^2_1$ periodogram
  that recovers the 16.35-day activity period of FRB 20180916B [@chime2020periodic]
  (`frbperiod`), and a uniform Catalog 2 repeater timing and lensed-delay census
  (`frbwait`, `frblens`);
- **Pulsars** — the steep mean radio spectrum and the $P$–$\dot P$ diagram from the
  ATNF catalogue (`pulsarspec`, `ppdot`), giant-pulse heavy-tail tests (`pte2`), and
  a monitoring-gap-robust glitch waiting-time classification (`glitchpop`);
- **HI & spectral line** — the flat inner Milky Way rotation curve from the
  Leiden/Argentine/Bonn survey [@kalberla2005] with modern Galactic constants
  [@reid2019] (`hi`), and an environment-split HI mass function from FASHI
  (`fashienv`);
- **Solar & heliospheric** — type III exciter-speed and beam-tracking analyses across
  e-Callisto, Wind/WAVES and STEREO/WAVES, including a four-instrument corona-to-0.4-AU
  synthesis (`solarbursts`, `windwaves`, `swaves`, `triangulate`, `type3synthesis`);
- **Planetary radio** — Jovian and Saturnian occurrence censuses (`junodam`, `skr`)
  and a blind reanalysis of the Voyager 2 PRA ice-giant rotation periods (`vgpra`);
- **Rotation measure, Faraday & cosmology** — the Galactic Faraday sky, RM structure
  functions, and the first RM dipole/isotropy test (`rmsky`, `rmstructure`,
  `rmdipole`);
- **Continuum & variability** — a VLASS [@lacy2020] multi-epoch variability census
  recovering FK Comae Berenices, multi-decade VLBI variability, image-plane stacking,
  Euclidean source counts, peaked-spectrum selection, radio–optical offsets, and RACS
  Stokes-V coherent-emitter searches (`vlass`, `vlbi`, `stacking`, `sourcecounts`,
  `peaked`, `southern`, `offsets`, `stokesv`);
- **SETI** — a seedable injection–recovery benchmark for narrowband Doppler-drift
  searches, complementary to signal-injection tools such as `setigen` [@brzycki2022]
  (`driftsearch`);
- **GPU / machine learning** — a device-portable, pure-PyTorch [@pytorch] Fast DM
  Transform and coherent-DSP suite that runs identically on CPU and on an AMD (ROCm)
  GPU (`fdmt`, `torchdsp`), and neural simulation-based inference [@sbi_toolkit] of a
  radio-emitter population (`svsbi`).

Each module exposes the same tested entry point to the command line, the notebooks,
and the orchestrators. Orchestration is matched to the data: static-input slices
build through a server-less Snakemake [@snakemake] file-target DAG (`make figures`),
while the frequently-updated e-Callisto archive is ingested by an Apache Airflow
[@airflow] DAG on rootless Podman. A `pull-station-data` bridge brings calibrated
bundles from the author's `jansky-observe` rooftop station into the same pipeline, so
self-collected HI spectra can join the public-archive slices. `make reproduce`
regenerates the whole chain — data, analyses, papers, and arXiv packages — from a
clean checkout.

# Statement of need

Open-source radio astronomy is well served for detection and I/O, but the
*statistical analysis* of public catalogues is typically reimplemented privately for
each paper, as one-off scripts that are rarely tested, packaged, or reproducible. A
newcomer or amateur who wants to take a public dataset and obtain its population
statistics, a rotation curve, a periodicity search, or a spectral-index sample — with
uncertainties and honest caveats — has no lightweight, installed library to do so,
one small enough to run on a laptop yet able to reach for a GPU when the
signal-processing warrants it. `jansky-research` fills that gap with a coherent,
tested toolkit aimed at three audiences: learners moving from a course to real data;
amateurs who want reproducible analyses without an observatory software stack; and
educators who need worked, citable examples that regenerate from source.

Its second contribution is methodological: a worked template for *honest*
small-scale research at scale. Every slice is put through an adversarial
science-review gate before write-up, and the project reports negatives and limits
rather than over-claiming. Two pillars recur across the whole toolkit. The first is
recover-a-known validation — each tool must recover an injected or historically
established signal (FRB 20180916B's period, the flat rotation curve, the Crab's DM,
published planetary rotation periods) before it is trusted on new data. The second is
an honest null: a large body of slices report bounded non-detections or systematics
rather than manufactured discoveries — the SETI detector fooled by a band-centre
DC-spike artifact (`driftsearch`), ultra-steep-spectrum candidates that are really the
TGSS flux-scale systematic (`spectra`), a blind Voyager 2 reanalysis that recovers
neither ice-giant period (`vgpra`), an OVRO-LWA type II census that is
false-positive dominated (`typeii`), and calibrated upper limits from megaconstellation
RFI trends, giant-pulse tests, repeater lensing, and Stokes-V emitter surveys
(`rfitrend`, `pte2`, `frblens`, `wdpulsar`). Not a single overclaim has
survived the gate. This makes the package useful not only as tooling but as a
reproducible reference for what careful, gate-checked amateur analysis of public data
can and cannot establish.

# Functionality and reproducibility

The package installs with `uv`/`pip`, depends on `jansky` for shared radio-astronomy
helpers, and exposes each slice as `python -m jansky_research.<slice>` with an
`--offline` flag selecting the synthetic fixture. The core install and continuous
integration are **CPU-only**: the GPU pieces are opt-in extras (`fdmt`, `sbi`) whose
`torch` resolves from the CPU wheel index for tests, while the accelerated legs run
from a ROCm virtual environment — the same pure-PyTorch code, no CUDA-specific
branches. Figures are emitted as vector PDFs and headline numbers as LaTeX macros, so
the AASTeX papers under `papers/` (one per slice) regenerate from the pipeline with no
hand-typed results. Code quality is enforced by `ruff`, `mypy`, and `pytest` with an
85% coverage floor on the synthetic fixtures, and releases follow Semantic Versioning
as codified in `VERSIONING.md`. A bundled `arxiv-submit` helper assembles and
validates an arXiv upload package per paper, and `pull-station-data` fetches the
rooftop station's observation bundles into the analysis pipeline.
